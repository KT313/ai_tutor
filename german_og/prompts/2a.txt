# Anleitung zur Erstellung von KI-Tutor-Lerninhalten

## 1\. Zielsetzung

Erstelle aus den bereitgestellten Dokumenten (z. B. ein Fragenkatalog und Vorlesungsfolien) eine einzelne JavaScript-Variable namens `CONTENT_DATA`. Diese Variable dient als strukturierte Wissensbasis für einen KI-gestützten Lern-Tutor.

## 2\. Grundprinzipien der Strukturierung

### A. Semantische Gruppierung statt linearer Abfolge

Die wichtigste Aufgabe ist es, die Inhalte aus den Dokumenten in wenige, thematisch zusammenhängende **Lernkarten** zu gruppieren. Erstelle **nicht** eine separate Karte für jede einzelne Frage aus dem Fragenkatalog. Fasse stattdessen logisch verwandte Themen zu einer übergeordneten Karte zusammen.

  * **Beispiel für gute Kartentitel:** "Grundlagen der Parallelverarbeitung", "Prozesse, Threads & Synchronisation", "Schwarmintelligenz-Algorithmen".
  * **Beispiel für eine schlechte Struktur:** Eine Karte für "Was ist ein Prozess?", eine weitere für "Was ist ein Thread?", eine dritte für "Was ist der Unterschied?". Fasse diese stattdessen in der Karte "Prozesse, Threads & Synchronisation" zusammen.

### B. Klare Trennung von Stichwort, Erklärung und Tutor-Wissen

Jeder einzelne Inhaltspunkt sollte nach folgendem Prinzip aufgebaut sein, um eine klare und interaktive Lernoberfläche zu schaffen:

1.  **`string` (Das Stichwort):** Dies ist der sichtbare Text. Er muss **extrem prägnant** sein – idealerweise ein bis drei Wörter. Er dient als Anker oder Label für das Konzept.

      * *Gut:* `"Prozess"`, `"Amdahlsches Gesetz"`, `"Vorteile"`
      * *Schlecht:* `"Ein Prozess ist ein Programm in Ausführung mit eigenem, isoliertem Speicherbereich."`

2.  **`tooltip` (Die Erklärung):** Dies ist die "erste Ebene" der Information, die bei Maus-Hover erscheint. Hier stehen die **kurze Definition, die wichtigste Formel oder eine prägnante Beispiel-Liste**. Halte den Tooltip informativ, aber kurz. HTML-Tags wie `<strong>` und `<br>` sind erlaubt und erwünscht.

3.  **`verbose_tutor_info` (Das Tutor-Wissen):** Dies ist die "zweite Ebene" der Information, die **nur für die KI sichtbar** ist. Hier gehören umfassende Erklärungen, alternative Definitionen, häufige Missverständnisse, Beispiele und spezifische Anweisungen für den Tutor hinein, um tiefgründige Fragen stellen zu können.

### C. Lesbarkeit der Lernkarte als oberstes Ziel

Die Anordnung der `string`-Elemente auf der finalen Webseite muss logisch und leicht lesbar sein.

  * **Listen mit Label:** Wenn eine Zeile ein Konzept und dessen Unterpunkte darstellt, sollte das erste `string`-Element als Label dienen und mit einem **Doppelpunkt** enden.

      * *Beispiel:* `[{ "string": "Thread-Organisationsformen:" }, { "string": "Dispatcher/Worker" }, ...]`
      * *Ergebnis:* • Thread-Organisationsformen: **Dispatcher/Worker** **Team** **Pipeline**

  * **Klar benannte Hauptthemen:** Statt vager Labels wie `"Eigenschaften:"` sollte das Hauptkonzept selbst als Label dienen.

      * *Schlecht:* `Thread Eigenschaften:`
      * *Gut:* `Thread` (wobei die Eigenschaften im Tooltip oder `verbose_tutor_info` stehen)

## 3\. Detaillierte JSON-Struktur

Die Grundstruktur bleibt wie in der ursprünglichen Anleitung, wird aber durch die oben genannten Prinzipien ergänzt.

```javascript
const CONTENT_DATA = [
  {
    // Dies ist eine thematische Lernkarte (z.B. "Prozesse, Threads & Synchronisation")
    "title": {
      "string": "Titel der übergeordneten Lernkarte",
      "tooltip": "Eine kurze Beschreibung des übergeordneten Themas der Karte.",
      "verbose_tutor_info": "Generelle Lernziele für diese Karte und Hinweise für den Tutor, worauf er sich konzentrieren soll."
    },
    "content": [
      // Jedes innere Array ist eine Zeile auf der Lernkarte.
      [
        // Jedes Objekt in der Zeile ist ein sichtbares Element (Stichwort).
        {
          "string": "Prägnantes Stichwort 1",
          "tooltip": "Kurze Definition oder Erklärung zu Stichwort 1.",
          "verbose_tutor_info": "Umfassende Details und Kontext zu Stichwort 1 für den Tutor."
        },
        {
          "string": "Prägnantes Stichwort 2",
          "tooltip": "Kurze Definition oder Erklärung zu Stichwort 2.",
          "verbose_tutor_info": "Umfassende Details und Kontext zu Stichwort 2 für den Tutor."
        }
      ],
      // Nächste Zeile auf der Karte
      [
        {
          "string": "Weiteres Konzept",
          "tooltip": "Erklärung zu diesem Konzept.",
          "verbose_tutor_info": "Zusatzwissen für den Tutor."
        }
      ]
    ]
  }
];
```
